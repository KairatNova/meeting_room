# Промпт для написания и сокращения текста пояснительной записки

Используй этот файл как **обязательную инструкцию** при создании или правке любых `.md` в папке `diploma/` (кроме `README.md`, `WORD_FORMAT.md` и этого файла). Цель — чтобы при переносе в Word блок **от «Введение» до «Заключение»** уложился в **55–65 страниц**.

---

## Промпт (скопировать ассистенту)

```
Ты редактируешь выпускную квалификационную работу (тема: веб-приложение для бронирования переговорных комнат).
Работай только с файлами в папке diploma/. Соблюдай diploma/PROMPT.md и diploma/WORD_FORMAT.md.

Целевой объём основного текста (Введение — Заключение): 55–65 страниц в Word при:
— шрифт Times New Roman, 14 пт;
— межстрочный интервал 1,5;
— абзац по ширине, красная строка 1,25 см (по методичке);
— поля: левое 30 мм, правое 15 мм, верх/низ 20 мм (уточни у кафедры).

Ориентир по объёму: ~10 000–12 000 слов на весь блок Введение…Заключение
(не считая список источников и приложения). Один раздел — компактный академический текст, без «воды».

Структура нумерации:
— нумеруются только разделы: «1.1. …», «2.3. …», «3.1. …»;
— подтемы БЕЗ подномеров: «Актуальность темы», «Состав схемы» — отдельной строкой, без «1.3.6.»;
— не использовать коды ФТ, НФТ, БП, UC; писать по-русски.

Оформление (WORD_FORMAT.md):
— без markdown-заголовков #, без **жирного**, без ---;
— таблицы: подпись СВЕРХУ «Таблица 1.1 – Название»;
— рисунки: подпись СНИЗУ «Рисунок 2.1 – Название»;
— ссылки на литературу: [1], [2] или [Источник №];
— URL в скобках: (https://example.com/).

Сжатие текста (обязательно):
— короткие абзацы (3–6 предложений), не более 2–3 абзацев под одной подтемой;
— списки требований и функций — в ТАБЛИЦАХ (компактно), не десятками отдельных абзацев «Обязательно: …»;
— не дублировать одно и то же между разделами (аналоги — в 1.2, стек — в 1.4, API — в 2.3, реализация — в 3.x);
— сравнение технологий — таблица + 1 абзац вывода, не отдельный подпункт на каждую технологию;
— листинги кода в основной текст НЕ вставлять (только в приложения).

Лимиты по разделам (ориентир для Word, стр.):
| Файл | Раздел | Целевой объём, стр. |
| 00-vvedenie.md | Введение | 1 |
| 1.1 | | 2–3 |
| 1.2 | | 3–4 |
| 1.3 | | 4–5 |
| 1.4 | | 4–5 |
| 2.1–2.4 | Раздел 2 | 12–14 (по 3–4 на подраздел) |
| 3.1–3.5 | Раздел 3 | 16–18 (по 3–4 на подраздел) |
| 04-zaklyuchenie.md | Заключение | 1 |
| Итого | | 55–65 |

При сокращении существующего текста приоритет урезания:
1) 1.3 и 1.4 (самые длинные);
2) повторы между 2.3 и 3.1, 2.4 и 3.3;
3) детальные перечисления эндпоинтов — одна сводная таблица на группу маршрутов.

Согласованность с проектом meeting_room: FastAPI, PostgreSQL, React, Telegram, Render (+ Railway как альтернатива в 3.5), nine tables in DB, verify-email not email SMTP for codes.

Выход: готовый текст в .md для копирования в Word, без служебных комментариев «здесь рисунок» внутри абзацев — только «рис. 2.1» в тексте.
```

## Почему в Word вышло 110+ страниц

Типичные причины при тех же TNR 14 / 1,5:

1. Каждая строка таблицы и каждый пункт «Обязательно:» вставились как **отдельный абзац** с интервалом до/после.
2. Лишние **пустые строки** после вставки из Markdown.
3. В основной текст попали **листинги кода** или черновик `DIPLOMA_DRAFT_V2.md`.
4. Широкие таблицы (§1.2, §1.3, §2.3) переносятся **по одной строке на страницу**.
5. Заголовки разделов с большим «интервалом до/после» (стили Word).

Список источников и приложения в лимит **55–65 стр. не входят** — только Введение … Заключение.

---

## Правила вставки в Word (чтобы уложиться в 55–65 стр.)

1. Вставлять **без сохранения исходного форматирования** (чистый текст), затем применить стиль «Основной текст».
2. Таблицы: вставить → преобразовать в таблицу Word → шрифт в таблице **12 пт**, межстрочный **1,0** в таблице.
3. Удалить пустые абзацы (включить отображение ¶).
4. Заголовки разделов: интервал до/после **не более 6 пт**.
5. Не вставлять код и скриншоты между §1.1 и §3.5 — только в приложения.

## Бюджет слов (контроль объёма)

| Блок | Слов (макс.) |
|------|----------------|
| Введение | 350 |
| Раздел 1 (1.1–1.4) | 3 200 |
| Раздел 2 (2.1–2.4) | 2 800 |
| Раздел 3 (3.1–3.5) | 3 600 |
| Заключение | 300 |
| **Итого** | **~10 250** |

Перед сдачей: в Word выделить только Введение…Заключение → **Рецензирование → Число слов**. Если **> 13 000 слов** — сокращать §1.3 и §1.4 в первую очередь.

## Задачи для ассистента по файлам (порядок сокращения)

1. Сократить `01-razdel-1/1.3-trebovaniya.md` и `1.4-tehnologicheskij-stek.md` до лимитов таблицы выше.
2. Проверить разделы 2 и 3 на повторы с разделом 1 — убрать дубли.
3. Не расширять введение и заключение (≤ 1 стр. каждое).
4. После правки — оценка: суммарно ~10 000 слов в файлах 00, 01-*, 02-*, 03-*, 04.

## Что не трогать без запроса

- `05-spisok-istochnikov.md`, `06-prilozheniya.md` — вне лимита 55–65 стр.
- `DIPLOMA_APPENDIX.md` в корне — только для приложения Б, не в основной текст.
- Факты о проекте (Telegram, nine tables, Render/Railway) — не выдумывать.
