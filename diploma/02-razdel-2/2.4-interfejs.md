2.4. Проектирование пользовательского интерфейса

Клиентская часть реализуется как одностраничное приложение (SPA): переходы между экранами выполняются без полной перезагрузки документа, данные запрашиваются у REST API (п. 2.3). Ниже описаны структура маршрутов, ключевые экраны и принятые решения по удобству использования.

Навигация и уровни доступа

Маршрутизация построена на React Router 6. Общая оболочка Layout содержит шапку с логотипом, ссылками на каталог комнат и информационные страницы, переключатель языка и меню пользователя (профиль, бронирования, админ-панель, выход).

Маршруты делятся на три группы (табл. 2.6).

Таблица 2.6 – Маршруты клиентского приложения

| Путь | Доступ | Назначение |
| --- | --- | --- |
| / | Публичный | Главная: дашборд для вошедшего или лендинг для гостя |
| /login, /register | Публичный | Вход и регистрация |
| /verify-email | Публичный | Ввод кода подтверждения после регистрации |
| /forgot-password, /reset-password | Публичный | Восстановление пароля |
| /rooms, /rooms/:id | Публичный | Каталог и карточка комнаты с календарём |
| /about, /help, /how-to-use | Публичный | Справочные страницы |
| /profile, /my-bookings, /my-bookings/:id | Авторизованный | Профиль и бронирования |
| /admin | Администратор | Управление комнатами и ролями |

Неавторизованный пользователь при обращении к защищённым путям перенаправляется на /login; пользователь без прав администратора — с /admin на главную. Компоненты PrivateRoute и AdminRoute инкапсулируют эту логику.

Ключевые экраны

Главная страница для авторизованного пользователя показывает ближайшие бронирования, быстрые ссылки и блоки «избранное» и «недавно просмотренные» (данные в localStorage). Для гостя отображается описание сервиса и призыв войти или зарегистрироваться.

Каталог комнат — список с фильтрами и сортировкой, согласованными с параметрами GET /api/rooms. Карточка комнаты — галерея фото, параметры, средняя оценка по отзывам, календарь FullCalendar с занятыми слотами (данные из GET /api/bookings) и форма создания брони. При конфликте времени клиент показывает сообщение об ошибке 409.

Раздел «Мои бронирования» — таблица или список собственных броней с переходом на страницу деталей. Там же — отмена с подтверждением; кнопка блокируется, если до начала осталось менее 30 минут (проверка на клиенте и на сервере).

Административная панель объединяет вкладки: редактирование справочника комнат (создание, правка, загрузка и удаление фотографий) и управление списком администраторов (назначение по email, снятие роли).

Связь с API и состояние клиента

Модуль api вызывает эндпоинты с базовым URL из переменной VITE_API_URL; JWT хранится в памяти контекста AuthContext и подставляется в заголовки запросов. Типы ответов согласованы с backend-схемами (файл types/api.ts).

Интерфейс переведён на русский язык по умолчанию; словарь вынесен в модуль i18n с возможностью переключения локали в шапке.

Решения по удобству использования

Для длительных загрузок применяются скелетоны (SkeletonBlocks), для пустых списков — карточки с пояснением (EmptyStateCard). Формы входа и регистрации блокируют повторную отправку на время запроса. Вёрстка на Tailwind CSS адаптируется к ширине экрана (мобильный телефон — настольный монитор).
