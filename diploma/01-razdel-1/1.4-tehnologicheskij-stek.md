1.4. Выбор и обоснование технологического стека

Стек выбирался по критериям: модель «клиент — REST — СУБД», открытые лицензии, скорость разработки одним исполнителем, автотесты и OpenAPI, бесплатный облачный хостинг. Сводный состав — в табл. 1.7.

Серверная часть

Python 3.11+ — язык серверной логики и тестов. FastAPI (≥ 0.109) на Uvicorn: валидация Pydantic 2, зависимости Depends, автогенерация /docs. По сравнению с Django REST Framework выбранный вариант компактнее для узкого API без дублирования React-админки; по сравнению с Flask не требует отдельной связки для схем и OpenAPI.

PostgreSQL 16 — транзакции, внешние ключи, индексы по времени броней. SQLAlchemy 2.0 + Alembic + psycopg2-binary — ORM и миграции. SQLite используется только в тестах Pytest.

Безопасность: bcrypt для паролей, python-jose (JWT, HS256). Коды регистрации и входа доставляются через Telegram Bot API (webhook), без SMTP.

Клиентская часть

React 18.2 и TypeScript 5.6 — SPA и типизация контрактов API. Vite 5 — сборка и dev-сервер с прокси /api. Tailwind CSS 3.4 — адаптивная вёрстка. React Router 6 — маршруты и PrivateRoute / AdminRoute. FullCalendar 6 — календарь занятости на карточке комнаты.

Альтернативы Vue/Angular на frontend и Node/Express на backend отклонены: избыточны для масштаба приложения; единый акцент курса — React + Python API.

Тестирование и развёртывание

Pytest и httpx — backend; Vitest и happy-dom — frontend. Production: Render (render.yaml) — PostgreSQL, Web Service (Uvicorn), Static Site (dist); переменные DATABASE_URL, SECRET_KEY, TELEGRAM_BOT_TOKEN, VITE_API_URL. Допускается аналогичный деплой на Railway.

Сравнение ключевых альтернатив

Таблица 1.7 – Обоснование выбора основных компонентов

| Компонент | Выбрано | Краткое сравнение с альтернативой |
| --- | --- | --- |
| API-фреймворк | FastAPI | Django/DRF — тяжелее; Flask — нет OpenAPI из коробки |
| СУБД | PostgreSQL | SQLite — слабее при конкурентной записи; MongoDB — лишняя для реляционной схемы |
| UI | React + TS | Vue/Angular — не требуются по учебному треку |
| Сборка | Vite | CRA — медленнее; Next.js — SSR не нужен |
| Auth | JWT + Telegram | Сессии в Redis — лишний сервис; OAuth — вне scope |

Таблица 1.8 – Состав технологического стека

| Уровень | Технология | Назначение |
| --- | --- | --- |
| Backend | Python 3.11+, FastAPI, Uvicorn | REST API |
| Данные | PostgreSQL, SQLAlchemy 2, Alembic | Хранение и миграции |
| Безопасность | bcrypt, python-jose | Пароли, JWT |
| Интеграция | Telegram Bot API | Коды, webhook |
| Frontend | React 18, TypeScript 5.6, Vite 5 | SPA |
| UI | Tailwind CSS 3.4, React Router 6, FullCalendar 6 | Вёрстка, навигация, календарь |

Связка Python (FastAPI) + PostgreSQL + React + Telegram + Render покрывает требования п. 1.3 и поддерживает проектирование (раздел 2) и реализацию (раздел 3).
