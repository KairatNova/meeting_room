# Презентация проекта (для не-IT)

Этот файл поможет человеку без технического опыта объяснить проект преподавателю.

## 1. Что это за проект (20-30 секунд)

**Meeting Rooms** — это сайт для бронирования переговорных комнат.

Пользователь может:
- зарегистрироваться и войти,
- найти подходящую комнату по фильтрам,
- посмотреть занятость в календаре,
- забронировать комнату на нужное время,
- видеть и отменять свои бронирования.

Администратор может добавлять и редактировать комнаты.

## 2. Какую проблему решает

Без системы люди часто:
- бронируют одну и ту же комнату на одно и то же время,
- не видят, какие комнаты свободны,
- теряют информацию о своих бронированиях.

Проект решает это автоматически: проверяет конфликты и показывает занятость заранее.

## 3. Основные функции (показать по экрану)

1. **Регистрация и вход**
   - есть подтверждение email кодом.

2. **Главная страница**
   - список комнат,
   - фильтры (дата, время, вместимость, удобства, локация),
   - переключение языков.

3. **Страница комнаты**
   - подробная информация,
   - календарь занятости,
   - создание бронирования,
   - отзывы.

4. **Мои бронирования**
   - список будущих/прошедших броней,
   - отмена будущей брони.

5. **Админ-панель**
   - добавление/редактирование/удаление комнат,
   - загрузка фотографий.

## 4. Что важно с точки зрения логики

Самая важная часть — защита от двойного бронирования:
- если выбранное время уже занято, система не даст создать бронь;
- пользователь получает понятное сообщение об ошибке.

Также есть ограничения по длительности:
- минимум 30 минут,
- максимум 6 часов.

## 5. Какие технологии использованы (простыми словами)

- **Frontend (интерфейс):** React + Tailwind  
  (то, что видит пользователь в браузере).

- **Backend (сервер):** FastAPI + PostgreSQL  
  (где хранится информация и обрабатываются запросы).

- **FullCalendar:** для отображения занятости по времени.

- **JWT-авторизация:** безопасный вход пользователя.

- **Alembic:** миграции базы данных (изменения структуры таблиц).

## 6. Что можно сказать как итог

Проект готов как рабочий MVP:
- закрывает полный цикл бронирования,
- поддерживает роли (пользователь/админ),
- имеет API-документацию,
- покрыт тестами на конфликты бронирования.

## 7. Короткий текст для защиты (можно читать дословно)

"Мы реализовали веб-приложение для бронирования переговорных комнат.  
Пользователь регистрируется, подтверждает email, входит в систему, выбирает комнату по фильтрам и бронирует время.  
На сервере реализована проверка конфликтов, поэтому двойное бронирование невозможно.  
Есть личный кабинет для управления своими бронями и админ-панель для управления комнатами.  
Проект сделан на React + FastAPI + PostgreSQL, с календарем занятости и API-документацией."
